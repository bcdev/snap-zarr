Export SNAP-ZARR

Export SNAP-ZARR Product

This command allows to export the data of the current product to SNAP-ZARR format.

SNAP-ZARR uses the JZarr framework, which is a Java implementation of the Python Zarr library.

Constraints of created SNAP-ZARR product files

The exported product consists of a root zarr group and zarr-sub-groups and/or zarr-arrays. The root node contains meta information related to the product. The child nodes can then be further groups or zarr arrays. Subgroups are used to group related data. Raster data are stored in zarr arrays.
An exported product is fully compatible and thus directly processable with the Python Zarr library.

  • Name and type of the product.
  • Name, description and unit of each band.
  • The properties of no-data and scaling for each band.
  • The index coding of a band.
  • Virtual and filtered bands.
  • etc.

For features and limitations of the SNAP-ZARR export and constraints of interaction with other GIS tools please see the following list:

  • All bands of a SNAP-ZARR file must have the same data type. Therefor a common data type is identified which fits for all bands. Double precision data type is not supported.
  • The data of a band is always written raw and unscaled.
  • Virtual and filtered bands can only be reimported by the Sentinel Toolbox.
  • All projections available in SNAP are also supported by the SNAP-ZARR writer. As a fallback, if a map projection or other geocoding is not directly supported a tie-point grid is written to the SNAP-ZARR file.
    The exported SNAP-ZARR files were tested for compatibility with ENVI and ArcGis. While the files are fully compatible with ENVI, ArcGIS has problems to understand stereographic projections.
  • An index coded band can only be correctly decoded by other GIS tools, if it is written as an image with one band.
